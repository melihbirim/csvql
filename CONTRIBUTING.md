# Contributing to csvq

Thank you for your interest in contributing to csvq! Contributions of all kinds are welcome — bug reports, performance improvements, feature implementations, and documentation fixes.

## How to Contribute

1. **Fork the repository** and create a feature branch from `main`.
2. **Make your changes**, ensuring existing tests pass (`zig build test`).
3. **Open a pull request** with a clear description of your change and why it is useful.

## Code Style

- Follow the existing Zig code style and project conventions.
- Keep changes focused and minimal — one logical change per pull request.
- Add or update tests when introducing new behavior.

## Reporting Issues

Please open a GitHub issue with a clear description, steps to reproduce, and (if applicable) sample CSV data that triggers the problem.
